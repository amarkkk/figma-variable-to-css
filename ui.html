? What are grid proportions?

Proportion variables represent fractional widths on a 12-column grid. Auto-detected by patterns like "half", "third", "quarter", "two-thirds", "three-quarters".

Output: Unitless column count (--prop-half: 6) and flex-ready value (--prop-half--fr: 6fr).

These are always applied automatically — no opt-in needed. Shown here for visibility.

? What is viewport-relative scaling?

Standard clamp() interpolates between fixed breakpoints. Viewport-relative variables use min(100vw, max), tying the value directly to the full viewport width.

When to use:

  • Container or layout widths that should expand to the full viewport
  • Max-width constraints like content wrappers or hero sections

Detection: Variables with "viewport" in their name or description are auto-detected.

üìà Piecewise Scaling 0 ‚ñº
? What is piecewise scaling?

Standard clamp() draws a single straight line between the smallest and largest breakpoint. Piecewise clamp creates separate scaling segments between each adjacent pair of breakpoints, following the exact values your design team set.

Good candidates:

  • Typography that needs aggressive scaling on small screens but gentle scaling on large screens
  • Spacing tokens where tablet values are intentionally held closer to desktop
  • Any value where intermediate breakpoints were deliberately tuned

How detection works: Every numeric variable with at least one value that differs across breakpoints is listed below. Variables are grouped by collection and path. Select individual variables or use the collection/group checkboxes for bulk selection.
